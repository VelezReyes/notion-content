The correct methodology
Follow these steps in order. Skipping context is the #1 reason presentations come out generic.
1

Set context before anything else

Tell Claude: who the audience is, what the goal of the deck is, the tone, and the brand rules. Claude has no memory between sessions — every new conversation starts blank. Give it the full brief.

→ Paste your brand-manual.md in the first message. Claude will apply it throughout.

2

Define structure before content

Ask Claude to confirm slide count, section names, and accent color choice before writing a single word of content. Changing structure later is expensive in tokens and effort.

→ Never jump straight to "generate slides" on the first message.

3

Feed content section by section

Give Claude the raw content for one section at a time. For each slide, specify: title, main message (one sentence), supporting points, any data, and the layout type.

→ Use Template 2 from the prompt pack.

4

Approve the outline before generating code

Have Claude output a structured slide outline first. Review it. Only after approval should Claude generate the .pptx code. This saves 5,000–10,000 tokens on rework.

5

Generate the PPTX file

Claude uses PptxGenJS to produce a working Node.js script, runs it in the sandbox, and outputs a downloadable .pptx. For large decks (20+ slides), split into two calls.

6

QA pass

Ask Claude to review: brand colors, text overflow, slide variety, speaker notes. Claude converts slides to images and inspects them. Fix in one dedicated pass.

→ Use Template 6 from the prompt pack for a structured QA.

Prompt structure anatomy
Every strong prompt has these five zones. Missing one drops output quality significantly.
R

Role

Tell Claude what it is. "You are a presentation designer helping me build a branded slide deck." Sets behavior, tone, and decision-making frame for the entire session.

C

Context

Brand manual, audience, goal, tone, chosen accent color. Everything Claude needs to make consistent decisions without asking. The more specific, the less back-and-forth.

I

Input

The actual content — slide titles, bullet points, data, quotes. Structured clearly. One section at a time is always better than dumping everything at once.

O

Output format

Tell Claude what to produce: "Output a numbered slide outline as a table", "Generate a PptxGenJS Node.js script", "Confirm structure then wait for content". Claude defaults to prose if you don't specify.

G

Guard

What NOT to do. "Do not generate slides yet." "Do not add placeholder text." "Do not change brand colors." Without guards, Claude fills gaps with reasonable-but-wrong defaults.

→ Guards are especially important for brand color consistency.

Minimal example — first message
You are a presentation designer. BRAND: [paste brand-manual.md] DECK: 15 slides for Q2 company review. Audience: executive team. Goal: show progress + set Q3 priorities. Tone: confident, data-driven. Primary accent for this deck: #27CF77 Green Do not generate slides yet. Confirm you understand, then ask for section content.
Giving Claude the content correctly
How you structure input determines output quality more than any other factor.
A

One main message per slide

Every slide must answer: "What is the ONE thing the audience must leave knowing?" Put it as the "Main message" in your input. If you can't write it in one sentence, the slide needs to be split.

B

Max 40 words on screen

Slides are not documents. If you have more than 40 words for a slide, move the rest to speaker notes. Tell Claude: "Max 40 words on screen. Rest goes to speaker notes."

C

Distributing heavy content

If you have a large amount of raw material, paste it all and ask Claude to distribute it: "Organize this into a slide-by-slide outline. Max 15 slides. Flag what should be stat callouts." Then review and approve before generating.

D

Specify visual intent

Don't leave layout up to Claude. For each slide say: "two-column layout", "large stat callout centered", "icon grid 3 columns". Claude will use it.

Slide content input example
SLIDE 3 — Content Background: Cream (#EFEFE4) Accent for this slide: #27CF77 Green Title: Revenue Performance H1 Main message: Revenue grew 40% YoY, driven by enterprise segment. On screen: - $4.2M total revenue (+40% YoY) - Enterprise: 68% of total (up from 51%) - SMB: stabilized at $1.3M Layout: Large stat callout "$4.2M" centered top, two supporting stats below. Speaker notes: Shift in enterprise mix driven by Q1 deal closures, not price increases.

Cost-saving strategies
1

Approve outline before generating

Generating and then restructuring is 3–5x more expensive than getting the outline right first. Never skip the outline approval step.

2

Split large decks

For 20+ slides, generate slides 1–10 first, then continue with 11–20 passing the existing file as context. Avoids context window overflow.

3

Keep brand manual concise

Your brand-manual.md goes in every message. Keep it under 800 words. The provided template is optimized for this.

Usage rules
DoPick one dominant accent per deck and carry it through cover, section dividers, and key callouts.
DoUse a second accent for contrast — e.g. Yellow primary + Green for positive data labels.
Don'tUse all three accents on the same slide — they compete and cancel out visually.
Don'tPut dark text on Purple or white text on Yellow/Green — contrast always wins.
The notepad method
Prepare prompts offline before pasting. The most efficient way to work with Claude on complex deliverables.
1

Open a plain text editor

Notepad, Bear, Notion, VS Code — anything. Build your prompt here before touching Claude. This prevents wasting tokens on incomplete thoughts.

2

Fill out Template 1 completely

Paste in your brand manual. Write the audience, goal, tone, section structure, and your chosen accent color for this deck. Don't skip fields — every blank is a place Claude will improvise.

3

Write all slide content in the same file

Use Template 2 structure for each slide. You now have a complete content brief offline — reviewable, shareable, revisable — before a single token is spent.

4

Paste in order: context → structure → sections → generate

One message at a time. Wait for Claude to confirm before moving to the next step.

→ brand-manual.md + presentation-prompt-templates.md are your ready-to-use starting point.

5

Save your prompt file — reuse it

For recurring decks (monthly reviews, board updates), keep the template and just update the content section. Context setup time drops to near zero for repeat formats.
